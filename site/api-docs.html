API Documentation

Public REST API for the WARBIRD ecosystem. Handles authentication, social connections, community features, and more.

Base URL https://warbird-api.azurewebsites.net
Rate Limiting: All endpoints are rate-limited. Excessive requests will receive a 429 Too Many Requests response.

Health

GET /api/health Check API status

Returns the current health status of the API and database connectivity.

Response

{
  "status": "healthy",
  "timestamp": "2026-03-03T12:00:00Z",
  "database": "connected",
  "users": 42
}

Status Codes

200 OK

Authentication

POST /api/auth/register Create a new account

Register a new user with username and password.

Request Body

ParameterTypeDescription
usernamestringrequiredUnique username (3-30 chars)
passwordstringrequiredPassword (min 6 chars)

Response

{
  "id": 1,
  "username": "warbird_fan",
  "createdAt": "2026-03-03T12:00:00Z"
}

Status Codes

201 Created 400 Bad Request 409 Conflict
POST /api/auth/login Log in

Authenticate with username and password.

Request Body

ParameterTypeDescription
usernamestringrequiredYour username
passwordstringrequiredYour password

Response

{
  "id": 1,
  "username": "warbird_fan",
  "twitter": null,
  "telegram": null
}

Status Codes

200 OK 401 Unauthorized

Twitter / X OAuth

GET /api/auth/twitter/authorize Start Twitter login

Initiates the Twitter OAuth 2.0 PKCE flow. Returns a URL the client should redirect to.

Response

{
  "authUrl": "https://twitter.com/i/oauth2/authorize?...",
  "state": "random_state_string"
}

Status Codes

200 OK
GET /api/auth/twitter/callback Twitter OAuth callback

Handles the callback from Twitter after user authorization. Exchanges the code for an access token, fetches user info, and redirects back to the frontend with user data.

Query Parameters

ParameterTypeDescription
codestringrequiredAuthorization code from Twitter
statestringrequiredState parameter for CSRF verification

Status Codes

302 Redirect 400 Bad Request

Telegram

}

Status Codes

200 OK
POST /api/auth/telegram/verify Verify Telegram login

Verifies the HMAC-SHA256 hash from the Telegram Login Widget to authenticate a user.

Request Body

ParameterTypeDescription
idstringrequiredTelegram user ID
first_namestringUser's first name
usernamestringTelegram username
photo_urlstringProfile photo URL
auth_datestringrequiredUnix timestamp of auth
hashstringrequiredHMAC-SHA256 verification hash

Response

{
  "verified": true,
  "user": {
    "id": "123456789",
    "username": "telegram_user",
    "first_name": "John"
  }
}

Status Codes

200 OK 401 Unauthorized
